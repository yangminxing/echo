# 同步物理表功能实现计划

## 概述
在表单管理列表页面增加"同步物理表"按钮，点击后将表单 JSON 解析并生成对应的 MySQL 物理表结构。

## 功能流程
1. 用户点击"同步物理表"按钮
2. 后端读取表单 JSON 配置
3. 解析 JSON 中的组件信息，提取字段名、字段类型、字段标签等
4. 根据 epic-designer 组件类型映射到 MySQL 数据类型
5. 生成 CREATE TABLE SQL 并执行
6. 更新表单状态为"已同步"

---

## 一、组件类型与 MySQL 类型映射

| Epic-Designer 组件类型 | MySQL 数据类型 | 说明 |
|----------------------|---------------|------|
| input | VARCHAR(255) | 输入框 |
| textarea | TEXT | 多行文本 |
| number | DECIMAL(20,4) | 数字输入 |
| select | VARCHAR(100) | 下拉选择 |
| radio | VARCHAR(100) | 单选框 |
| checkbox | VARCHAR(500) | 多选框，存储逗号分隔值 |
| date | DATE | 日期选择 |
| date-range | VARCHAR(100) | 日期范围，存储两个日期 |
| time | TIME | 时间选择 |
| switch | TINYINT(1) | 开关 |
| rate | INT | 评分 |
| slider | INT | 滑块 |
| upload | VARCHAR(500) | 文件上传，存储文件路径 |
| rich-text | LONGTEXT | 富文本编辑器 |

## 二、后端实现

### 2.1 创建 DDL 工具类
新建 `src/main/java/org/echo/util/FormSchemaParser.java`:
- `parseSchema(String formSchema)` - 解析 JSON Schema
- `generateCreateTableSQL(String tableName, String formSchema)` - 生成建表 SQL
- `getComponentFieldInfo(JSONObject component)` - 提取组件字段信息

### 2.2 创建同步结果实体
在 `FormDefinition` 实体中添加字段：
- `tableName` - 物理表名
- `syncStatus` - 同步状态（未同步/已同步）
- `syncTime` - 同步时间

### 2.3 更新 Service
修改 `FormDefinitionService.java`:
- `syncToPhysicalTable(String id)` - 同步到物理表

### 2.4 更新 Controller
修改 `FormDefinitionController.java`:
- `POST /form-definition/sync/{id}` - 同步物理表接口

### 数据库更新
```sql
ALTER TABLE echo_form_definition 
ADD COLUMN table_name VARCHAR(100) COMMENT '物理表名' AFTER form_schema,
ADD COLUMN sync_status VARCHAR(20) DEFAULT '未同步' COMMENT '同步状态' AFTER table_name,
ADD COLUMN sync_time DATETIME COMMENT '同步时间' AFTER sync_status;
```

## 六、JSON Schema 解析示例

Epic-Designer 的 Schema 结构：
```json
{
  "schemas": [
    {
      "type": "page",
      "id": "root",
      "children": [
        {
          "type": "input",
          "field": "username",
          "label": "用户名",
          "props": { "placeholder": "请输入用户名" }
        },
        {
          "type": "number",
          "field": "age",
          "label": "年龄",
          "props": { "defaultValue": 0 }
        }
      ]
    }
  ]
}
```

生成的 SQL：
```sql
CREATE TABLE `form_用户表` (
  `id` VARCHAR(36) PRIMARY KEY,
  `username` VARCHAR(255) COMMENT '用户名',
  `age` DECIMAL(20,4) COMMENT '年龄',
  `create_time` DATETIME COMMENT '创建时间',
  `update_time` DATETIME COMMENT '更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```
